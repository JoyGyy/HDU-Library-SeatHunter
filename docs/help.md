# SeatHunter 使用帮助

## 概念说明

### 方案（Plan）

方案是预约的基本单位，每个方案包含以下信息：

- 方案 ID：用于标识方案，如 `morning_A42`（添加时自定义）
- 房间名和楼层名：座位所在位置
- 开始时间：格式为 `HH:MM:SS`，如 `08:00:00`（仅时间，不含日期）
- 使用时长：单位为小时
- 座位号：可包含多个座位

> 注意：方案的开始时间只是时间模板，不含日期。实际预约时由调度器自动填入目标日期。

### 调度（Schedule）

调度定义"什么时候执行哪些方案"，有两种模式：

- **按星期**：每个工作日自动预约，如每周一至周五
- **按日期**：指定具体日期预约，如 2026-05-20

调度通过 `plan_ids` 绑定到具体方案，一个调度可绑定多个方案。

### 预约触发规则

图书馆规定：**目标日期 X 的座位在 X-2 天的 20:00 开放预约**。

例如：想预约 5 月 20 日（周三）的座位，系统会在 5 月 18 日（周一）20:00 自动发起预约。

## 主菜单功能说明

### 1. 查看/添加/删除座位方案

管理你的预约方案库。

**添加方案步骤：**
1. 选择房间类型（系统会自动加载可用房间列表）
2. 选择楼层
3. 输入开始时间（`HH:MM:SS` 格式，如 `08:00:00`）
4. 输入使用时长（小时）
5. 输入座位号（多个用逗号隔开）
6. 输入方案 ID（用于标识，如 `morning_A42`）

**删除方案：** 输入方案序号，多个用逗号隔开。

### 2. 批量修改方案预约时间

同时修改多个方案的时间。输入序号时用逗号隔开，输入 `0` 表示修改全部方案。

> 警告：错误的时间可能导致封号一周的惩罚，请仔细核对。

### 3. 立即开始抢座

使用当前已有的方案立即发起预约请求。系统会按方案逐个尝试，每个方案重试直到成功或达到最大重试次数。

适用于：现在马上就要抢座位的场景。

### 4. 启动/管理定时调度

这是核心功能，支持以下操作：

| 子选项 | 说明 |
|--------|------|
| 启动自动调度 | 读取配置文件中的调度表，启动后台调度引擎 |
| 停止自动调度 | 停止正在运行的调度引擎 |
| 添加按星期调度 | 设置每周几自动预约（1=周一，7=周日），并绑定方案 |
| 添加按日期调度 | 设置具体日期预约，并绑定方案 |
| 查看已保存的调度 | 展示所有调度及绑定的方案详情 |
| 删除调度 | 按序号删除某个调度 |

**启动调度后：**
- 调度引擎在后台运行，主菜单仍可正常使用
- 系统会显示实时倒计时和下次触发时间
- 到达触发时间后自动执行预约

### 5. 查看调度状态

显示调度引擎的实时状态：
- 下次触发时间
- 距离下次触发还剩多少时间
- 将要执行的方案信息

### 6. 修改请求间隔和次数

| 设置项 | 说明 | 建议 |
|--------|------|------|
| 重试间隔 | 两次请求之间的等待秒数 | 不小于 5 秒，过短可能被封号 |
| 最大重试次数 | 单次预约最多尝试多少次 | 默认 10 次 |

### 7. 使用帮助

打开本帮助文档。

### 8. 退出

安全退出程序。如果调度引擎正在运行，会自动停止后再退出。

## 两种使用方式

### 交互模式（默认）

```
python main.py
```

进入交互式菜单，手动操作所有功能。适合首次配置和不定期使用。

### 守护模式

```
python main.py --daemon
```

读取配置文件后直接启动调度引擎，无交互菜单。适合无人值守运行（如 nohup、systemd、开机自启）。

使用守护模式前，需要先通过交互模式完成：
1. 登录账号
2. 添加方案
3. 添加调度

## 配置文件

配置文件位于 `config/config.yaml`，可以在交互模式下通过菜单操作，也可以直接编辑。

### 典型配置示例

```yaml
user:
  login_name: "你的学号"
  password: "你的密码"
  org_id: "104"

plans:
  - id: "morning_A42"
    room_name: "自习室A"
    floor_name: "1楼"
    begin_time: "08:00:00"
    duration_hours: 4
    seats:
      - seat_id: "12345"
        seat_num: "042"

schedules:
  - mode: weekdays
    target_weekdays: [1, 2, 3, 4, 5]
    plan_ids: ["morning_A42"]
    enabled: true

settings:
  interval: 5
  max_try_times: 10
  auto_relogin: true
```

### V1 配置自动迁移

如果你之前使用过旧版本，首次启动时会自动检测旧格式配置并迁移，原文件备份为 `config.yaml.v1.bak`。

## 常见问题

**Q: 登录失败怎么办？**
- 确认已连接 HDU 校园网或 VPN
- 检查学号密码是否正确
- 网络问题会自动重试 5 次

**Q: 预约失败提示"已被预约"？**
- 座位已被他人预约，建议准备多个备选方案

**Q: 会被封号吗？**
- 请求间隔设为 5 秒以上可避免，系统默认 5 秒

**Q: Cookie 会过期吗？**
- 登录缓存有效期为 20 天，过期后自动重新登录（需确保密码已保存）

**Q: 如何取消正在运行的调度？**
- 交互模式：菜单选择 4 → 停止自动调度
- 守护模式：按 Ctrl+C 或发送终止信号
