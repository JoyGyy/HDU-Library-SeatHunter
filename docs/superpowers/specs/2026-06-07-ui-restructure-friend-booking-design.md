# UI 重构 + 好友代预约设计

> 日期: 2026-06-07
> 状态: 已批准

## 背景

当前 GUI 有 5 个 Tab（预约、调度、工具、设置、帮助），存在以下问题：
- 功能分散：方案、调度、签到分布在不同 Tab，用户不清楚操作流程
- 缺少引导：新用户打开软件不知道该干什么
- 好友代预约需要手动操作：用户每次需要手动告诉朋友去同意预约
- 签到逻辑不够透明：用户不确定能否代签好友座位

## 设计目标

1. 重构 Tab 结构，让功能分组更直觉
2. 添加新手引导，首次使用时逐步完成初始配置
3. 实现好友代预约全自动化（预约 + 同意 + 签到）
4. 保持签到逻辑不变（已验证：一个 bookingId 覆盖所有座位）

---

## 1. Tab 重构

新结构：4 个 Tab

| Tab | 标题 | 内容 |
|-----|------|------|
| 0 | **首页** | 新手引导（首次使用）/ 运行状态仪表盘（日常使用） |
| 1 | **预约** | 方案管理 + 调度引擎 + 签到（一体化） |
| 2 | **好友** | 好友资料管理 + 代预约配置 + 自动同意设置 |
| 3 | **设置** | 账号信息 + 请求参数 + 帮助文档 |

### 变更说明

- 原"预约"+"调度"合并为新"预约"Tab（方案→调度→签到是同一流程的三个阶段）
- 原"工具"Tab 中 UID 管理移入"好友"Tab，预约历史移入"预约"Tab
- 原"帮助"Tab 内容移入"设置"Tab 底部
- 新增"首页"Tab（引导 + 仪表盘）
- 新增"好友"Tab（好友管理 + 代预约）

## 2. 首页：新手引导 & 仪表盘

### 2.1 首次使用（引导模式）

检测条件：无配置文件 或 无账号 或 无方案时自动进入引导模式。

引导步骤：

**Step 1：登录**
- 输入学号 + 密码
- 点击"登录"→ 调用 Playwright 自动登录
- 成功后显示 UID 和姓名
- 底部按钮：下一步

**Step 2：添加好友**（可跳过）
- 输入好友学号 + 密码
- 点击"查询"→ 调用 `lookup_uid` 获取好友 UID
- 显示好友姓名 + UID
- 可添加多个好友
- 底部按钮：跳过 / 下一步

**Step 3：创建方案**
- 选择房间 → 选择座位（支持选两个座位：自己 + 好友）
- 设置开始时间、持续时长
- 如果 Step 2 添加了好友，自动勾选"代预约好友"并填充好友 UID
- 底部按钮：下一步

**Step 4：设置调度**（可跳过）
- 选择调度类型（每天 / 每周指定天 / 指定日期）
- 设置触发时间（几点几分抢座）
- 底部按钮：跳过 / 完成

完成后自动切换到仪表盘模式。

## 3. 好友系统

### 3.1 好友资料管理

存储方案：复用 `config/uids.json`，新增 `friends` 字段：

```json
{
  "uids": {
    "23051110": {"uid": "303687", "name": "*睿智"},
    "23140322": {"uid": "305033", "name": "*3140322"}
  },
  "friends": {
    "23140322": {
      "uid": "305033",
      "name": "*3140322",
      "student_id": "23140322",
      "password_base64": "xxx"
    }
  }
}
```

- `password_base64`：密码用 base64 编码存储（防君子不防小人，本地桌面应用安全级别足够）
- 好友必须先通过 `lookup_uid` 查询 UID，查询时自动保存密码

### 3.2 代预约配置

在"预约"Tab 的添加方案对话框中：
- 如果已有好友，显示"代预约好友"勾选框
- 勾选后自动添加第二个座位，`booker_uid` 设为好友的 UID
- 方案列表中显示预约人信息（如 "100(我) + 101(好友)"）

### 3.3 自动同意

新增 `seathunter/services/friend_service.py`：

```python
class FriendService:
    """好友代预约服务"""

    def __init__(self, friend_store: FriendStore):
        self.store = friend_store

    def auto_confirm(self, booking_id: str, friend_student_id: str) -> tuple[bool, str]:
        """用好友账号自动确认预约

        1. 从 store 获取好友凭证
        2. 用 Playwright 以好友身份登录（独立 session）
        3. 调用 /Seat/Index/confirmBooking?bookingId=xxx
        4. 返回 (成功?, 消息)
        """
```

预约成功后的完整流程：

```
book_seat 成功 → 拿到 booking_id
    ↓
如果有好友代预约:
    FriendService.auto_confirm(booking_id, friend_student_id)
    ↓
register_checkin(booking_id, ...)  # 注册自动签到
    ↓
history.log(...)  # 记录到历史
```

- 自动同意失败不影响主预约和签到
- 失败只记录日志 + UI 提示

## 4. 签到逻辑（无需改动）

已验证结论：
- 签到 API 只看 `bookingId`，不看登录账号
- `book_seat` 传入多个座位时，返回一个 `bookingId` 覆盖所有座位
- 用登录账号签到该 `bookingId`，所有座位都生效
- 自动签到窗口：预约开始时间前 25 分钟到后 25 分钟

## 7. 不做的事情

- 不改签到逻辑（已验证 OK）
- 不改预约 API 调用逻辑（book_seat 已支持多人）
- 不做真正的密码加密（base64 混淆够用）
- 不做远程通知机制（朋友在本地管理即可）
