# 网页版完整功能设计

## 概述

为 HDU Library SeatHunter 网页版（部署在 Railway）实现完整功能，包括方案管理、调度管理、引擎控制，使其能完全替代桌面版。

## 目标

- 网页版覆盖 100% 后端 API 功能
- 手机浏览器可完成所有操作
- 保持现有深色主题风格

## 布局

底部 5 个 Tab 导航：

| Tab | 图标 | 功能 |
|-----|------|------|
| 预约 | 📋 | 当前预约列表 + 签到 |
| 方案 | 📝 | 创建/查看/删除预约方案 |
| 调度 | ⏰ | 定时任务 + 引擎控制 |
| 好友 | 👥 | 好友管理 |
| 设置 | ⚙️ | 退出登录 |

## 功能设计

### 1. 方案管理 Tab

**列表显示：**
- 卡片列表，每个方案显示：房间名、楼层、座位列表、开始时间、时长
- 左滑或长按删除（网页用删除按钮）

**创建方案（Modal 弹窗）：**
- 步骤 1：选择房间（下拉框，从 API 获取房间列表）
- 步骤 2：选择楼层（下拉框，根据房间联动）
- 步骤 3：选择座位（列表，支持多选）
- 步骤 4：设置开始时间和时长
- 保存按钮

**API 调用：**
- `GET /api/plans` — 获取方案列表
- `POST /api/plans` — 创建方案
- `DELETE /api/plans/{id}` — 删除方案
- `GET /api/rooms` — 获取房间列表（需要新增）
- `GET /api/rooms/{id}/seats` — 获取座位列表（需要新增）

### 2. 调度管理 Tab

**引擎状态卡片：**
- 显示：运行状态（运行中/已停止）、下次触发时间、倒计时
- 启动/停止按钮
- 每 5 秒自动刷新

**调度列表：**
- 卡片列表，每个调度显示：关联方案名、触发时间、状态（启用/禁用）
- 删除按钮

**添加调度（Modal 弹窗）：**
- 选择关联方案（下拉框）
- 选择触发类型：每天 / 指定日期
- 设置触发时间
- 保存按钮

**API 调用：**
- `GET /api/schedules` — 获取调度列表
- `POST /api/schedules` — 添加调度
- `DELETE /api/schedules/{id}` — 删除调度
- `POST /api/schedules/start` — 启动引擎
- `POST /api/schedules/stop` — 停止引擎
- `GET /api/schedules/status` — 获取引擎状态

### 3. 预约 Tab（已有）

保持现有功能不变。

### 4. 好友 Tab（已有）

保持现有功能不变。

### 5. 设置 Tab

- 显示当前登录用户
- 退出登录按钮

## 技术方案

### 前端

- 纯 HTML/CSS/JS，无框架
- 复用现有深色主题（Catppuccin Mocha）
- Modal 弹窗用于创建方案和调度
- 轮询机制更新引擎状态

### 后端

- 需要新增房间和座位查询 API（如果不存在）
- 复用现有 plans、schedules 路由

### 文件结构

```
server/static/
├── index.html      # 主页面（更新 Tab 结构）
├── style.css       # 样式（新增方案和调度相关）
└── app.js          # 逻辑（新增方案和调度功能）
```

## 验收标准

1. 可以创建预约方案（选房间、楼层、座位、时间）
2. 可以查看和删除方案
3. 可以添加定时调度
4. 可以启动/停止调度引擎
5. 引擎状态实时显示
6. 所有功能在手机浏览器可用
